# Бриф задачи — Alfa Bank x MFTI «Отклик на кредитный оффер»

## Цель

Предсказать вероятность того, что корпоративный клиент примет конкретный кредитный оффер. Итоговая модель должна ранжировать офферы по вероятности принятия. В PDF указано, что один запрос клиента может получить несколько возможных офферов, поэтому валидация должна проверять структуру повторяющихся запросов/клиентов/офферов, а не считать строки независимыми.

## Данные

Ожидаемые файлы платформы:

- `train_apps.csv` — обучающие данные с `target_value`;
- `test_apps.csv` — тестовые данные без `target_value`;
- `sample_submission.csv` — требуемая схема сабмита.

Обучающие данные включают параметры заявки, условия оффера, характеристики клиента и признаки финансовой активности.

## Таргет и метрика

- `target_value = 1` — оффер принят;
- `target_value = 0` — оффер отклонён;
- метрика: ROC-AUC.

## Репрезентативные предикторы

Колонки ниже — примеры из PDF, а не полный закрытый список. Необходимо изучить фактическую схему train/test и каждую обнаруженную колонку либо использовать, либо трансформировать, либо явно исключить с указанием причины.

- условия оффера: `loan_amount_last`, `overdraft_limit_min`, `overdraft_limit_max`, `offered_rate`, `cb_rate`;
- время: `decision_day`;
- банковская активность: суммы и количества дебета/кредита в окнах 30/90/360 дней;
- ликвидность/баланс: `balance_rur_amt_30_min`, инвестиционные суммы;
- кредитная история: активные продукты, кредитные счётчики, месяцы с момента кредитных сделок;
- цифровое поведение: события дашборда, время в системе;
- категориальные: `db_group_last`, `fl_adminarea`.

## Ограничения

Использовать Python 3.10+ и open-source библиотеки. Использовать только предоставленные train/test данные, если дополнительные файлы не разрешены явно. Построить валидный CSV-сабмит и избегать лишних загрузок на платформу. Не более 3 загрузок на платформу в день. Для контекста проекта ChatGPT Plus держать пакет агентов в пределах 25 файлов и не включать документы миграций/changelog/истории изменений.
