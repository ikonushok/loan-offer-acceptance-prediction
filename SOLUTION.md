# Описание решения — Отклик на кредитный оффер (Альфа-Банк × МФТИ)

## 1. Постановка задачи

Задача — предсказать вероятность того, что корпоративный клиент примет предложенный кредитный оффер (`target_value = 1` — принял, `0` — отказался). Данные в плоском формате: одна строка = один оффер (`front_id`), 26 сырых признаков (ставка, лимиты, активность клиента за 30/90/360 дней, категории, `decision_day`, `cb_rate`). Метрика — ROC-AUC. Дисбаланс классов: ~6.1% акцептов. **Train/test разделены по времени:** train февраль 2024 – июнь 2025, test июнь – декабрь 2025 (99.9% теста — за временной границей train).

Итоговый публичный результат: **ROC-AUC = 0.76744** (76.744).

Разведочный анализ данных с визуализациями — в [EDA-ноутбуке](notebooks/eda_final.ipynb).

## 2. Архитектура решения

Решение построено на **rank-бленде двух семейств GBM с календарной осью времени**, где главный рычаг — признак `decision_day_num` (линейный счётчик дней), дающий модели позицию во времени.

```
                          ┌─────────────────────────────────┐
                          │   Исходные данные (плоские)     │
                          │  145k train / 36k test строк    │
                          │  26 признаков + decision_day    │
                          └──────────┬───────────┬──────────┘
                                     │           │
            context-offer +          │           │   context-offer +
            КАЛЕНДАРЬ (day_num+week) │           │   КАЛЕНДАРЬ (day_num+week)
                    ▼                            ▼
          Представление 1:                Представление 2:
          CatBoost champion               XGBoost HPO
          (3 trial бленд)                 (depth=3, child=80, reg=30)
          ─────────────────               ─────────────────
          t070×0.70 + t041×0.04           pointwise binary
          + t164×0.26                     lh_mean 0.7678
          lh_mean 0.7622
                    │                            │
                    └──────────────┬─────────────┘
                                   ▼
                          RANK-BLEND (late-holdout отбор)
                          champion_nm 0.30 + xgb_nm 0.70
                          ─────────────────────────────
                          lh_mean = 0.7692 → public 0.76744
```

### 2.1. Ветка CatBoost (champion)

Бленд трёх Optuna-trial с разной регуляризацией (диверсификация внутри семейства):
- `trial0070` (depth 6, l2 4.7) × 0.70;
- `trial0041` (depth 5, l2 35) × 0.04;
- `trial0164` (depth 7, l2 16) × 0.26;
- `SqrtBalanced` class weights, raw-probability бленд.
- Признаки: context-offer (относительный ранг оффера в группе) + **календарь no_month**.

### 2.2. Ветка XGBoost (диверсификатор)

- `depth=3, min_child_weight=80, reg_lambda=30, lr=0.025, 1400 деревьев`, early stopping.
- Те же признаки + календарь no_month. Нативная обработка NaN и категорий.
- Декоррелирует с CatBoost (другое семейство) при сопоставимой силе.

### 2.3. Календарный сигнал — ключевой рычаг

Главное открытие проекта. Из `decision_day` извлекаются:
- `decision_day_num` — линейный счётчик дней от начала train (**ось времени**);
- `decision_week` — неделя года (маргинальный вклад);
- `decision_month` категориальный **исключён** — он переобучается (тестовые месяцы не встречаются в train).

`day_num` даёт модели понимание временного дрейфа (prevalence акцептов нестационарна 5×). На паблике добавление календаря дало +0.117 → +0.239 (76.388 → 76.505 → 76.744).

### 2.4. Ансамбль

Rank-percentile blend: предсказания каждой модели ранг-нормализуются (0..1) на тестовом наборе, затем взвешиваются. Веса подобраны сканом по late-holdout батарее (не по лидерборду). ROC-AUC — ранговая метрика, поэтому rank-нормализация корректна и инвариантна к масштабу probabilities CatBoost vs XGB.

Финальные веса: **champion_no_month 0.30 + xgb_no_month 0.70**.

## 3. Ключевые решения и обоснования

### Почему календарь (day_num), а не просто фичи?

Тест на 100% в будущем относительно train. Сильный временной дрейф: prevalence акцептов скачет 5× (0.03 → 0.155 → 0.07), связь фича→таргет распадается со временем. `day_num` даёт модели ось времени, позволяя калибровать ранжировку под поздний (тестоподобный) режим. Ablation показал: `day_num` даёт ~93% всего календарного эффекта.

### Почему `month` исключён?

Категориальный месяц переобучается на конкретные train-месяцы; тестовые месяцы (июль–декабрь 2025) — для него unseen-категории. На late-holdout `month` ухудшал метрику (−0.0004). Линейный `day_num` переносится, категориальный `month` — нет.

### Почему rank-blend, а не raw-probability?

CatBoost (`SqrtBalanced`) и XGB дают вероятности в разных масштабах. Rank-нормализация приводит обе к единой шкале перцентилей. Подтверждено эмпирически: rank-blend дал весь прирост 76.054 → 76.388 на стадии диверсификации.

### Почему late-holdout, а не Fold3, для отбора?

Fold3 (внутри train-диапазона) систематически недооценивает временные признаки. Календарь ухудшал Fold3, но улучшал late-holdout (будущеподобные H1/H2/H3) и паблик. **Урок: отбирать признаки для drift-задачи нужно по late-holdout, а не по in-range CV.**

### Почему отбор офлайн, а не LB-probing?

Эмпирически установлено: офлайн-порядок late-holdout предсказывает публичный (усиление ~27×). Это позволило выбирать кандидатов по lh_mean/lh_min без расходования загрузок на разведку.

## 4. Результаты

### Прогресс публичного ROC-AUC

| веха | модель | Public |
|---|---|---|
| baseline | CatBoost champion (без времени) | 0.76054 |
| диверсификация | + XGB rank-blend (w=0.49) | 0.76388 |
| **календарь** | XGB no_month (day_num+week) | 0.76505 |
| **финал** | full_champion_no_month × 0.30 + xgb_no_month × 0.70 | **0.76744** |

### Ablation календаря (XGB late-holdout, база без времени = 0.7633)

| признак | lh_mean | вклад |
|---|---|---|
| day_num (ось времени) | 0.7675 | +0.0042 (~93%) |
| + week | 0.7678 | +0.0003 |
| month (категориальный) | 0.7629 | −0.0004 (вредит) |

### Временной дрейф

| метрика | значение |
|---|---|

| prevalence (фев'24 / дек'24 / весна'25) | 0.031 / 0.155 / 0.07 |

| adversarial AUC train-vs-test (cb_rate) | 0.63 |

## 5. Известные ограничения

1. **GBM не экстраполирует тренд:** `day_num` на тесте упирается в границу train (clamp) — модель дотягивает до последнего train-режима, но не продолжает тренд. Линейная компонента/детрендинг — открытое направление.

2. **Late-holdout слеп к зиме:** покрывает только весну 2025; систематически недооценивает сезонные признаки. Для них нужен winter-holdout (обучение без ноя-дек'24, валидация на них).

3. **Циклическая сезонность не использована:** sin/cos(day-of-year) на winter-holdout помогает зимнему пику (+0.0027), но late-holdout штрафует; net-эффект на паблике не проверен (попытки закончились). Кандидат собран как резерв.

4. **Калибровка не проводилась:** ROC-AUC ранговая, калибровка не влияет на скор.

5. **Разрыв с лидером (0.789):** ~0.022 над финалом — вероятно структурный/внешний рычаг, не найденный текущим подходом.

## 6. Закрытые направления (проверены, отрицательны)

| направление | вердикт |
|---|---|
| HPO (CatBoost ×2, LGBM, XGB) | потолок Fold3 ≈ 0.755 |
| rate/limit ratios | temporal drift (cb_rate нестационарен) |
| missing-flags | GBM едят NaN нативно, флаги переобучают |
| adversarial importance weighting | ухудшает всё, теряет данные |
| pseudo-labeling | confirmation bias, ранжировку не двигает |
| TabNet (NN) | OOF 0.6955, слишком слаб для бленда |
| learning-to-rank (rank:pairwise) | декоррелирует, но OOF 0.695 |
| split feature space (absolute/context) | декорреляция и сила взаимоисключающи |
| multi-seed XGB | seed-вариативность ничтожна (corr 0.993) |

## 7. Открытые направления

| направление | ожидаемый эффект | сложность |
|---|---|---|
| Циклическая сезонность (публичная проверка) | неизвестно (winter-holdout +) | низкая |
| Детрендинг / линейная компонента по времени | средний (экстраполяция тренда) | средняя |
| Фазы cb_rate, дни до/после смены ставки | неизвестно | низкая |
| time_regime в LGBM (3-е семейство) | +0.001–0.002 | низкая |
| Разрешённый внешний источник / связка данных | большой (разрыв с лидером) | высокая |

- **Конфиги:** `configs/feature_experiments/catboost_trial00{70,41,64}_no_month_v1.json`, `configs/xgb_hpo_experiments/xgb_hpo_depth3_child80_reg30_v1.json` (+ no_month time_features).

- **Финальный сабмит:** `submissions/upload_20260621/candidate_20260621_upload3_full_champion_no_month_c30_x70.csv` (SHA256 `b238d426…45391bd`).
